About the Brand Concierge: Hackathon Edition

The **Brand Concierge AI** was developed to eliminate the "where is that file?" and "what's the hex code?" friction that plagues design and marketing teams. This project demonstrates a working **AI-grounded knowledge base** delivered in a responsive, modern web app.

Technical Execution:

  • **Knowledge Base:** Team guidelines are sourced from the **knowledge_base.txt** file (simulating a live API feed).
  • **Gemini 2.5 Flash API:** Used for hyper-efficient **structured JSON output** (reply, status, and link), ensuring reliable data parsing on the frontend.
  • **Empathy Prompting:** The system instruction guides the AI to reply with a warm, supportive, and conversational tone, turning a static document into a friendly assistant.
  • **Proxy Architecture:** A Vercel **Node.js Proxy** securely handles the API key and file reading, ensuring stable deployment even with complex environment constraints.

Future Roadmap: What Comes Next

Given more time, these are the immediate features we would integrate to turn this hackathon project into a production-ready internal tool.

Key Enhancements:

  • **Image Understanding:** Allow users to upload a design and ask, "Does this color match the brand guidelines?" The AI would verify colors/fonts in the image against the knowledge base.
  • **Live Source Integration:** Replace the static knowledge_base.txt file with a direct, automated fetch from **Figma, Notion, or Confluence** via their APIs.
  • **Google Search Grounding Re-integration:** Fix the structural issue preventing the use of the Google Search tool, allowing the AI to answer general questions (e.g., "latest accessibility standards") without relying solely on the internal document.
  • **Multi-User Authentication:** Implement secure login for designers, marketers, and developers, potentially allowing different knowledge bases or permissions for each group.
